UXUI audit pack

Alana UXUI visual testing checklist

Usa este checklist para ejecutar la revision visual del workbench operator-facing. El objetivo no es solo detectar bugs de UI, sino validar jerarquia visual, lectura, consistencia de copy, densidad operativa, manejo de estados y calidad responsive.

Fecha base
2026-03-10
Entorno principal
mock local
Entorno secundario
staging solo para spot-check visual
Breakpoints obligatorios
1440, 1280, 1024, 390
Temas obligatorios
light y dark
Evidencia minima
screenshot completo, hallazgo, severidad, impacto y recomendacion
Regla central: un mismo surface no debe mezclar espanol e ingles, no debe filtrar wording tecnico y no debe repetir el mismo resumen en varias capas si no agrega valor nuevo.
Severity

Criterio de severidad

P1

Bloquea lectura, confianza o accion.

P2

Degrada fuerte la comprension, velocidad o confianza.

P3

Inconsistencia o deuda visual no bloqueante.

Output

Entregables exigidos a UX/UI y Producto

  • Inventario de hallazgos por surface con severidad, impacto y recomendacion.
  • Lista separada de correcciones visuales: jerarquia, spacing, tipografia, contraste, CTA y estados.
  • Lista separada de correcciones de copy y microcopy.
  • Recomendacion explicita de que capas deben fusionarse, simplificarse o desaparecer.
  • Decision conjunta sobre direccion visual antes de una nueva iteracion frontend.
